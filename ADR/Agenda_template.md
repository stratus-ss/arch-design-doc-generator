## Post-Meeting Actions

After each session:

1. **Fill in the client ADR** — Update Decision, Status, Assumptions, and Argument fields for each discussed ADR
2. **Update the Weekly Status** — Add entries to the current week's table in `ADR_Weekly_Status.md`
3. **Identify next session topics** — Note which ADRs advanced to Partial and need follow-up
4. **Capture action items** — Document who owns what and by when

## Topic Selection Guidelines

- **Priority order:** Decisions that block installation > decisions that block design > nice-to-haves
- **Dependencies:** If ADR X depends on ADR Y, discuss Y first
- **Readiness:** Only bring topics where the right stakeholders are present
- **Batch related topics:** e.g., ADRs 27/28/29 (namespace + RBAC + placement) discuss together
- **Aim for 3–5 topics per 60-minute session** — better to close 3 than partially discuss 7
